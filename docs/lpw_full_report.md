## 3. 연구자님의 직관에 대한 객관적 숙고

### 직관 1: "Ellipse threshold를 튜닝하면 더 개선될 수 있다"

> **판정: ✅ 동의, 그러나 전략적으로 접근할 필요 있음**

정량적 근거가 이를 명확히 뒷받침합니다. 현재 하이브리드 Ellipse의 크기 임계값(5%)과 거리 임계값(50px)은 감으로 설정한 초기값이며, 이 값들이 모든 폴더에 일률적으로 적용되고 있습니다. 그런데 결과를 보면:
- **Baseline에서 낮았던 폴더(0.30~0.50대):** 대부분 대폭 향상 → 현재 threshold가 효과적
- **Baseline에서 높았던 폴더(0.70 이상):** 소폭 하락 가능 → 현재 threshold가 과도하게 개입

이는 하나의 고정 threshold로는 모든 case를 커버할 수 없음을 뜻합니다.

**"LPW fALL 1 epoch로 threshold를 최적화"**하자는 제안에 대해: 이것은 grid search로 쉽게 구현 가능합니다. 면적 비율 threshold를 {0.03, 0.05, 0.10, 0.15}, 거리 threshold를 {30, 40, 50, 60}으로 잡으면 16개 조합이며, 이미 만들어진 스크립트 구조를 활용하면 충분히 실행 가능합니다.
다만, **Folder 7, 17처럼 이미 깨끗한 예측에서 후처리가 역효과를 낳는 패턴**은 threshold 조정만으로는 근본적으로 해결이 어려울 수 있습니다. 이 경우 "Ellipse를 아예 적용하지 않는 것이 이 프레임에서는 최적"인데, 이를 자동으로 판단하는 로직(예: 동공 예측의 원형도/완결도가 이미 높으면 Ellipse를 Skip)이 필요합니다.

### 직관 2: "속눈썹 부근에서 시멘틱 붕괴 → 블러 세기를 높이면 더 무너짐 → 학습 가중치를 건드려야"

> **판정: ✅ 완전히 동의**

σ sweep 실험(0.5~3.0)에서 σ=1.0 이후 **단조감소**가 확인되었으므로, 현재 Skip Filter 방식으로는 σ=1.0이 이론적 상한선입니다. 속눈썹 분절의 근본 원인은 학습 도메인(OpenEDS 근적외선)에서 속눈썹이 거의 보이지 않아, 모델이 **"속눈썹을 관통하는 동공"이라는 시멘틱 자체를 학습한 적이 없다**는 점입니다. 이것은 주파수 영역 조작으로는 해결 불가능한 **시멘틱 갭(semantic gap)**이며, 해결하려면:
- 소량의 속눈썹 포함 데이터로 ViT encoder의 최상위 몇 레이어만 fine-tuning (LoRA 등)
- 또는 속눈썹에 대응하는 data augmentation을 적용한 re-training

이들은 모두 학습 가중치를 건드리는 방식이므로 **Future Work으로 분류하는 것이 적절**합니다.

### 직관 3: "광량 과소(동공 확대) 프레임에서 더 무너진다 → 학습 문제"

> **판정: ✅ 동의, 흥미로운 관찰**

직관적으로는 과노출(washout)이 더 치명적일 것 같지만, LPW 데이터에서 **저광량 환경에서 동공이 극도로 확장된 프레임**은 OpenEDS 학습 분포(HMD 내부의 일정한 근적외선 조명)와 극심하게 괴리됩니다. OpenEDS의 동공은 비교적 일정한 크기인 반면, 저광량 LPW에서 동공이 홍채 전체를 뒤덮을 정도로 커지면 모델이 이를 "동공"으로 인식하지 못합니다. 과노출 케이스는 동공 신호 자체가 물리적으로 소실되므로 해결 불가능하나, 저광량의 경우는 신호(동공 경계)가 존재하므로 **학습으로 회복 가능**합니다. 이 역시 Future Work입니다.

### 직관 4: "안경알이 눈을 가로질러지면 무너진다"

> **판정: ✅ 동의**

TransUNet이 학습한 OpenEDS 데이터셋은 HMD 내부의 근적외선 카메라로 촬영된 것이므로 안경 프레임이나 렌즈 반사가 존재하지 않습니다. 안경 테두리가 동공을 가로지르면 이것은 모델이 학습한 적 없는 **완전히 새로운 occluder 패턴**이며, Skip Filter로 고주파를 감쇠해도 안경 테두리의 저주파 성분(굵고 뚜렷한 직선)은 감쇠되지 않습니다. 이것은 확실한 Future Work이며, occlusion-aware segmentation 기법이 필요합니다.

### 직관 5: "반쯤 감긴 눈에서 Ellipse가 오히려 결과를 열화시킨다"

> **판정: ✅ 핵심적 통찰, 구조적 한계로 기록해야 함**

이것은 하이브리드 Ellipse 모듈의 **본질적 한계(Structural Limitation)**입니다. `cv2.fitEllipse`는 입력 점군에 대해 **무조건 닫힌 타원**을 피팅합니다. 그러나 반쯤 감긴 눈에서 동공이 눈꺼풀에 의해 잘려 반달 모양일 때, 닫힌 타원은 눈꺼풀 아래 영역까지 포함하는 **위양성**을 만들어냅니다.

정량적으로 Folder 7, 17의 하락 원인에도 이 패턴이 기여했을 가능성이 높습니다. 해결책은 두 가지입니다:
1. **Confidence 기반 Ellipse Skip:** 모델 예측의 원형도(circularity)나 blob 점유율이 이미 충분히 높으면 후처리를 적용하지 않음
2. **Open-curve fitting:** 타원 대신 반달 형태를 허용하는 피팅 (구현 복잡도 높음)

연구자님 말씀대로, 반눈 케이스 자체가 trivial한 상황이므로 이에 과도하게 최적화하는 것은 경계해야 하며, **"모듈의 한계점"으로 논문에 명시**하는 것이 적절합니다.

## 6. Grid Search 실험 결과 (Ellipse Threshold 최적화 + Confidence-aware Skip)

### 6.1 실험 설계

§5에서 제안했던 두 가지 추가 실험을 통합하여 **단일 Grid Search 파이프라인**으로 구현했습니다.

| 항목 | 값 |
|------|-----|
| **탐색 파라미터** | 면적 비율 {0.03, 0.05, 0.10, 0.15} × 거리 {30, 40, 50, 60} × 원형도 Skip {0.80, 0.85, 0.90, 1.00} |
| **총 조합 수** | 64개 |
| **평가 대상** | LPW 전체 22개 폴더, 66개 비디오, ~132,000 프레임 |
| **실행 방식** | Dual GPU (RTX 3090 ×2) 분산 처리. Folder 1~11(GPU0), 12~22(GPU1). 순전파 1회 캐싱 후 후처리만 64회 반복. |
| **소요 시간** | 약 1시간 10분 |

**Confidence-aware Ellipse Skip의 원리:** 가장 큰 동공 Contour의 **원형도(Circularity = 4πA/P²)**를 실시간으로 계산합니다. 이 값이 `conf_threshold` 이상이면, 예측이 이미 충분히 깨끗한 것으로 간주하고 Ellipse 후처리를 건너뛰어 과개입에 의한 열화를 방지합니다. `conf_threshold=1.00`은 사실상 "항상 후처리 적용"(Skip 비활성화)을 의미합니다.

### 6.2 전체 결과 — 64개 조합의 Total mIoU 분포

### 6.3 핵심 발견: 후처리 파라미터 공간이 극도로 평탄(Flat)

Grid Search의 가장 중요한 발견은, **64개 조합 간 성능 차이가 최대 0.71%p에 불과하다**는 것입니다. 이는 다음을 의미합니다:

1. **후처리 파라미터 튜닝의 한계:** 현재 파이프라인의 성능 병목은 후처리(Ellipse)가 아닌 **딥러닝 모델의 시멘틱 예측 품질** 자체에 있습니다. 후처리는 이미 주어진 예측을 다듬는 역할만 하므로, 아무리 정교하게 튜닝해도 상한선이 존재합니다.
2. **수동 초기값의 적절성:** 감으로 설정했던 초기값(a=0.05, d=50)이 최적값(a=0.10, d=50)과 0.0001 차이에 불과하여, 초기 직관이 사실상 최적에 근접했습니다.
3. **최적 파라미터:** `면적 비율 0.10, 거리 50px`이 가장 안정적. 기존 0.05 대비 면적 비율을 2배로 올린 것이 미세하게 유리합니다. 작은 파편을 더 공격적으로 제거하는 편이 낫다는 뜻입니다.

### 6.6 Grid Search 실험 결론

2. **Confidence-aware Skip은 현재 구현으로는 무효합니다.** 원형도 기반 판별이 실제 하락 프레임(반눈, 속눈썹 침투)의 특성을 포착하지 못합니다.
3. **성능의 상한선은 모델의 시멘틱 예측 품질에 의해 결정됩니다.** 후처리를 넘어선 개선을 위해서는 학습 가중치 수준의 개입이 필요합니다.

## 7. 추가 실험 제안 (12~16시간 예산)

Grid Search 결과가 명확히 보여주듯, **후처리 파라미터 튜닝은 포화**되었습니다. 남은 12~16시간의 실험 시간을 가장 효과적으로 쓸 수 있는 방향을 우선순위 순으로 제안합니다.

### 제안 1: Skip Filter σ의 Skip 위치별 세분화 탐색 (예상 2~3시간)
현재는 Skip #0와 #1에 동일한 σ=1.0을 적용하고 있습니다. 그러나 Skip #0(해상도 높음, 고주파 풍부)과 Skip #1(해상도 낮음)은 전달하는 정보의 성격이 다르므로, **각 Skip에 독립적인 σ를 부여**하면 추가 이득을 얻을 가능성이 있습니다.
- 탐색: σ₀ ∈ {0.5, 1.0, 1.5, 2.0}, σ₁ ∈ {0.0, 0.5, 1.0, 1.5} = 16조합
- 기존 Grid Search 스크립트 구조를 재활용하여 빠르게 실행 가능.

### 제안 2: RITnet 전처리 기법 이식 (예상 1~2시간 구현 + 1시간 실험)
이전에 논의했던 RITnet 논문의 데이터 전처리(감마 보정 γ=0.8, CLAHE)를 추론 시 입력 이미지에 적용하여 광량 변동에 대한 강건성을 확보합니다. 재학습 없이 입력 단에서 도메인 갭을 줄이는 시도입니다.
- 추론 직전에 감마 보정 + CLAHE를 순차 적용하는 전처리 파이프라인 추가.
- Skip Filter + Ellipse와 조합하여 22개 폴더 전체 평가.

### 제안 3: Morphological Kernel 크기 탐색 (예상 2~3시간)
현재 9×9 MORPH_CLOSE 커널이 고정되어 있습니다. 커널 크기가 분절 연결 효과와 경계 왜곡 사이의 트레이드오프를 결정하므로, {5, 7, 9, 11, 13} 범위를 탐색하면 추가 개선 가능성이 있습니다.
- Grid Search 스크립트에 커널 크기를 파라미터로 추가하여 실행.

### 제안 4: Swirski 데이터셋에 Grid Search 최적 파라미터 교차 검증 (예상 30분)
LPW에서 찾은 최적 파라미터(a=0.10, d=50)가 Swirski에서도 동일하게 최적인지, 아니면 데이터셋마다 최적값이 다른지 확인합니다. 논문에 cross-dataset 일반화의 추가 근거가 됩니다.

### 우선순위 정리

| 순위 | 항목 | 예상 시간 | 기대 효과 |
